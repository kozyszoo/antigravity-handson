## このステップのゴール

MCP と Skills を組み合わせた本格的な Vibe Coding を体験します。Web リサーチから実装、コードレビュー、デプロイ準備まで一気に通してみましょう。

:::message
所要時間は約30分です。Step 1〜5 がすべて完了していること、Browser MCP が有効になっていることが前提です。
:::

## 基礎編との違い

| 項目 | Step 3（基礎編） | Step 6（発展編） |
|:---|:---|:---|
| MCP | なし | Browser, Filesystem を使用 |
| Skills | なし | カスタム Skill を使用 |
| リサーチ | なし | 自動 Web リサーチ |
| 複雑さ | 単一ページ | 複数ページ + API 連携 |
| 自動化 | 手動指示 | ワークフロー自動実行 |

## ハンズオン: AI Coffee Shop LP を作ろう

全体の流れはこうなっています。

```
Web リサーチ → 構成決定 → ビジュアル生成 → 実装 → 品質チェック → デプロイ
```

### Part 1: Web リサーチ（10分）

MCP の Browser 機能で競合サイトを調べます。

```
Browser MCPを使って、「おしゃれなカフェ ランディングページ」で検索して、
上位5サイトのデザインの特徴を分析して。

特に以下の点をまとめて：
- 色使い
- レイアウト
- CTAボタンのデザイン
- 画像の使い方
```

エージェントがブラウザを操作し、各サイトのスクリーンショットを取得して、デザインの特徴をレポートとして出力します。

### Part 2: 構成決定（5分）

リサーチ結果を踏まえて、LP のセクション構成を決めます。

```
リサーチ結果を踏まえて、このLPに必要なセクション構成を提案して。

要件:
- ターゲット: 20-30代のコーヒー好き
- ブランドイメージ: 近未来的 × 温かみ
- ゴール: 来店予約
```

### Part 3: ビジュアル生成（5分）

Nano Banana でヒーロー画像とアイコンを生成します。

```
Nano Banana で、「近未来的なカフェの店内、
ネオンブルーとウォームライト、観葉植物、
4K、フォトリアリスティック」のイメージを生成して。
ファイル名は hero-bg.webp で保存して。
```

### Part 4: 実装（5分）

プロジェクトを作成し、コンポーネントを生成します。

```
この 06_vibe_coding_advanced フォルダに、
Vite + React + TailwindCSS のプロジェクトを作成して。
プロジェクト名は「ai-coffee-shop」
```

```
以下のコンポーネントを作成して:
- Hero.jsx - ヒーローセクション
- Features.jsx - 特徴紹介（3つのカード）
- Menu.jsx - メニュー一覧
- Access.jsx - アクセス情報（地図埋め込み）
- CTA.jsx - 来店予約ボタン
- Footer.jsx - フッター

デザイン要件:
- グラスモーフィズムスタイル
- ダークモード
- アクセントカラー: #4ecca3
```

### Part 5: 品質チェック（3分）

code-review Skill でコードを自動チェックします。

```
@code-review ai-coffee-shop/src/components/ のすべてのコンポーネントを
レビューして、改善点があれば自動的に修正して。
```

レスポンシブ対応も確認しましょう。

```
以下の画面サイズでプレビューして、問題があれば修正:
- モバイル: 375px
- タブレット: 768px
- デスクトップ: 1280px
```

### Part 6: デプロイ準備（2分）

```
プロダクションビルドを実行して、
Firebase Hostingにデプロイする準備をして。
```

## 開発時間の比較

| 項目 | 従来の開発 | 基礎編 | 発展編 |
|:---|:---:|:---:|:---:|
| リサーチ | 2時間 | - | 10分 |
| デザイン | 4時間 | 20分 | 15分 |
| 実装 | 8時間 | 30分 | 20分 |
| レビュー | 2時間 | - | 5分 |
| **合計** | **16時間** | **50分** | **50分** |

基礎編と発展編の合計時間は同程度ですが、発展編ではリサーチとコードレビューが自動化されるため、より複雑なプロジェクトに対応できるようになります。

## 高度なテクニック

### MCP 活用例

```
「〇〇のAPIドキュメントを読んで、使い方をまとめて」
「〇〇.comのデザインを参考に、同じようなレイアウトで作って」
```

### Skills 活用例

```
@generate-component Button 「プライマリボタン、青色」
@code-review src/components/
@generate-docs README.md を作成
```

### マルチエージェント連携

```
以下のタスクを並列で実行して:
1. Designer Agent: UIコンポーネントのデザイン
2. Coder Agent: ロジックの実装
3. Researcher Agent: 最新のベストプラクティス調査
```
