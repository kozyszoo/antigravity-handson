## このステップのゴール

プロジェクト固有のルールをエージェントに教える設定ファイルを作成します。

## Rules とは

Rules は、エージェントの振る舞いを制御するための持続的な指示です。コーディングスタイル、応答言語、プロジェクトの技術スタックなどを定義しておくと、エージェントが常にそれに従って動きます。

たとえば「常に日本語で応答する」「コードにはコメントをつける」「変数名は camelCase にする」といったルールを書いておけば、毎回指示しなくても守ってくれるようになります。

:::message
公式ドキュメントはこちら: https://antigravity.google/docs/rules-workflows
:::

## 設定ファイルの種類

ルールを書くファイルには3つの名前が使えます。機能はすべて同じです。

- `GEMINI.md`（もっとも一般的）
- `AI_INSTRUCTIONS.md`
- `ANTIGRAVITY.md`

## 配置場所と適用範囲

ルールファイルはプロジェクト内のさまざまな場所に置けます。置いた場所によって影響範囲が変わります。

| レベル | パス | 適用範囲 |
|:---|:---|:---|
| グローバル | `~/.gemini/GEMINI.md` | すべてのプロジェクト |
| ワークスペース | `.agent/rules/` フォルダ内 | そのワークスペースのみ |
| プロジェクトルート | `GEMINI.md` | そのプロジェクト全体 |
| サブディレクトリ | 各フォルダ内の `GEMINI.md` | そのフォルダ以下のみ |

## Activation Mode

Rules には4つの発動モードがあります。

| モード | 動作 | 使いどころ |
|:---|:---|:---|
| **Always On** | 常に有効 | コーディング規約、言語設定 |
| **Manual** | `@ファイル名` で明示的に呼び出す | 特定のレビュー基準 |
| **Model Decision** | AIが説明文から自動判断する | 関連ルールの自動適用 |
| **Glob** | ファイルパターンに一致時に発動 | `src/**/*.ts` 向けのルールなど |

## ハンズオン: GEMINI.md を作成しよう

### ファイルを確認する

プロジェクトルートの `GEMINI.md` を開いてください。ハンズオン用リポジトリには、すでにサンプルが含まれています。

### 基本テンプレート

GEMINI.md の基本的な構成はこのようになっています。

```markdown
# プロジェクト設定 - Google Antigravity Hands-on

## 基本ルール
- 常に**日本語**で応答してください。
- コードには必ず**コメント**を入れてください（日本語可）。
- エラーが発生した場合は、原因と解決策を提示してください。

## 技術スタック
- HTML / CSS / JavaScript（Vanilla）
- Node.js (任意)

## コーディング規約
- 変数名: camelCase
- 関数名: camelCase
- ファイル名: kebab-case（例: `my-component.js`）
```

### 動作を確認する

チャットで以下を試してみましょう。

```
「Hello World」を表示する HTML ファイルを作成して。
```

ルールが効いていれば、以下のようになるはずです。

- 日本語で応答される
- コードにコメントが入っている
- 変数名が camelCase になっている

## 応用: ルールを追加してみよう

`GEMINI.md` に以下を足してみてください。エージェントの振る舞いが変わるか確認してみましょう。

```markdown
## 追加ルール
- ファイル作成時は必ず確認を求めてください。
- 長いコードは分割して説明してください。
```

## Workflows にも触れてみよう（任意）

Workflows は、繰り返しのタスクを手順書としてまとめて自動実行する仕組みです。`.agent/workflows/` フォルダに `.md` ファイルを置いて、チャットで `/ワークフロー名` と入力すると実行されます。

```markdown
# .agent/workflows/hello.md

---
description: 挨拶のテスト用ワークフロー
---

1. ユーザーに「こんにちは！何をお手伝いしましょうか？」と挨拶する
2. 現在の日付と時刻を表示する
3. 現在のディレクトリ構造を簡単に説明する
```

実行するときはチャットに `/hello` と入力するだけです。


## 日本語化を徹底するテクニック（Globalルールの活用）

毎回プロジェクトごとに `GEMINI.md` を書くのが面倒な場合や、うっかり忘れて英語で返されてしまう場合は、**Globalルール**を設定するのがおすすめです。

### Globalルールの設定手順

1. エージェントパネル右上の「…」メニューから `Customizations` を選択
2. `Rules` タブにある `Global` のプラスボタン（または編集ボタン）をクリック
3. 開いたファイルに以下のようなルールを記述して保存

```markdown
# 基本設定
回答は常に日本語で行ってください。
実装計画書（Implementation Plan）やタスクリストも日本語で作成してください。
```

これで、新しく作るすべてのプロジェクトで最初から日本語ルールが適用されます。

### それでも英語で出力されたら？

Antigravity は英語ネイティブな思考をするため、たまに英語で計画書やタスクリストを出してくることがあります。
その場合は焦らず、チャットでこう指示してください。

```
計画書を日本語に翻訳して更新して
```

AI が計画書を `Update` して日本語に書き直してくれます。内容を確認し、日本語になったことを確認してから `Proceed` ボタンを押せば OK です。
