## 目標
プロジェクト固有のルールをエージェントに教える設定ファイルを作成しましょう。

---

## Rules とは

Rules は、エージェントの振る舞いを制御するための持続的な指示です。

簡単に言うと、**「AIへの取扱説明書」**のようなものです。「日本語で答えて」「コードにコメントを入れて」などのルールを書いておくと、AIが毎回それに従って動いてくれます。

> 公式ドキュメント: https://antigravity.google/docs/rules-workflows

### 設定ファイルの種類

| ファイル名 | 説明 |
|:---|:---|
| `GEMINI.md` | プロジェクトルートに配置します。もっとも一般的です |
| `AI_INSTRUCTIONS.md` | 別名です。機能は同じです |
| `ANTIGRAVITY.md` | 別名です。機能は同じです |

### 配置場所と適用範囲

| レベル | パス | 適用範囲 |
|:---|:---|:---|
| **グローバル** | `~/.gemini/GEMINI.md` | すべてのプロジェクトに適用 |
| **ワークスペース** | `.agent/rules/` フォルダ内 | そのワークスペースにのみ適用 |
| **プロジェクトルート** | `GEMINI.md` | そのプロジェクト全体に適用 |
| **サブディレクトリ** | 各フォルダ内の `GEMINI.md` | そのフォルダ以下にのみ適用 |

### Activation Mode（ルールの発動条件）

Rules には 4つの発動モードがあります。※ 初心者は「**Always On**」だけ覚えればOKです。

| モード | 説明 | 例 |
|:---|:---|:---|
| **Always On** | 常に有効。設定しておけば勝手に適用される | コーディング規約、言語設定 |
| **Manual** | `@ファイル名` で呼び出したときだけ有効 | 特定のレビュー基準 |
| **Model Decision** | AIが「このルールが必要そう」と判断したときに自動適用 | 関連性のあるルール |
| **Glob** | 特定のファイルパターンに一致したときに発動 | `*.ts` のファイルのときだけ適用 |

## ハンズオン: GEMINI.md を作成しよう

### Step 1: ファイル作成

プロジェクトルートの `GEMINI.md` を開いて確認してください。すでに作成済みです。

### Step 2: 基本テンプレート

GEMINI.md の基本的な構成はこのようになっています。**普通のマークダウンファイル**と同じ書き方でOKなので、特別な文法を覚える必要はありません。

```markdown
# プロジェクト設定 - Google Antigravity Hands-on

## 基本ルール
- 常に**日本語**で応答してください。
- コードには必ず**コメント**を入れてください（日本語可）。
- エラーが発生した場合は、原因と解決策を提示してください。

## 技術スタック
- HTML / CSS / JavaScript（Vanilla）
- Node.js (任意)

## コーディング規約
- 変数名: camelCase
- 関数名: camelCase
- ファイル名: kebab-case（例: `my-component.js`）
```

### Step 3: 動作確認

Antigravity のチャットで試してみましょう。

```
「Hello World」を表示する HTML ファイルを作成して。
```

確認すること:
- 日本語で応答されているか
- コードにコメントが入っているか
- 変数名が camelCase になっているか

## 応用課題

### 課題 1: ルールを追加してみよう

`GEMINI.md` に以下を追加して、エージェントの振る舞いが変わるか確認してみましょう。

```markdown
## 追加ルール
- ファイル作成時は必ず確認を求めてください。
- 長いコードは分割して説明してください。
```

### 課題 2: Workflows を体験しよう（任意）

Workflows は、繰り返しのタスクを自動化する仕組みです。定型作業を「レシピ」として保存し、コマンド一つで実行できます。

1. `.agent/workflows/` フォルダに `.md` ファイルを作成します
2. チャットで `/ワークフロー名` と入力して実行します

```markdown
# .agent/workflows/hello.md

---
description: 挨拶のテスト用ワークフロー
---

1. ユーザーに「こんにちは！何をお手伝いしましょうか？」と挨拶する
2. 現在の日付と時刻を表示する
3. 現在のディレクトリ構造を簡単に説明する
```

実行方法:
```
/hello
```

### 日本語化を徹底するテクニック（Globalルールの活用）

毎回「日本語で」と書くのが面倒な場合は、**Globalルール**を設定することをおすすめします。

1. エージェントパネル右上の「…」メニューから `Customizations` > `Rules` > `Global` を選択
2. `gemini.md`（設定ファイル）に以下を記述して保存

<!--
  [図4b: Global Rules 設定画面]
  【スクリーンショット指示】
  Agent Manager のグローバルルール設定画面のキャプチャ。

  表示要素:
    1. 右上の「…」（三点メニュー）をクリックした状態でドロップダウンが開いている状態
       - 「Customizations」メニューがハイライトされている
    2. その下に Rules > Global 設定画面が開いている状態
       - エディタ同様にGEMINI.mdの内容が表示されている
    3. 画面内に緑色の矢印で「ここをクリック」と記載した注釈を付ける
  サイズ: 800×500px。縁取り付き。
-->

```markdown
# 基本設定
回答は常に日本語で行ってください。
実装計画書（Implementation Plan）やタスクリストも日本語で作成してください。
```

これで、すべてのプロジェクトで最初から日本語が適用されるようになります。

### もし英語で出力されたら？

設定していても英語で出力されることがあります。その場合は焦らず、チャットでこう指示してください。

```
計画書を日本語に修正して
```

AI が計画書を `Update` して日本語に書き直してくれます。
