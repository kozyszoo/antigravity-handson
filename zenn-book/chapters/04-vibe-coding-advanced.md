---
title: "Step 4: Vibe Coding 発展編"
---

MCP と Skills を組み合わせた、本格的な Vibe Coding を体験します。

## 学習目標

- MCP を活用した Web リサーチの実践
- Agent Skills を使った自動化フロー
- 複数エージェントの連携
- 本格的な Web アプリケーションの作成

## 所要時間

30分

## 前提条件

- Step 1〜3 がすべて完了している
- MCP が設定済み（Browser MCP は必須）
- Agent Skills の作成方法を理解している

> ⚠️ **React や TailwindCSS を知らなくても大丈夫です！**
> このステップでは React / Vite / TailwindCSS といったフレームワークを使いますが、**すべて AI がセットアップとコード生成を行います**。フレームワークの知識がなくても、Vibe Coding で「作りたいものを伝える」だけで進められます。

---

## 基礎編との違い

| 項目 | Step 2（基礎編） | Step 4（発展編） |
|:---|:---|:---|
| **MCP** | なし | Browser, Filesystem を使用 |
| **Skills** | なし | カスタム Skill を使用 |
| **リサーチ** | なし | 自動 Web リサーチ |
| **複雑さ** | 単一ページ | 複数ページ + API 連携 |
| **自動化** | 手動指示 | ワークフロー自動実行 |

## Part 1: Web リサーチ（MCP 活用）【10分】

### 1.1 競合サイトのリサーチ

MCP の Browser 機能を使って、競合サイトを調べます。

```
Browser MCPを使って、「おしゃれなカフェ ランディングページ」で検索して、
上位5サイトのデザインの特徴を分析して。

特に以下の点をまとめて：
- 色使い
- レイアウト
- CTAボタンのデザイン
- 画像の使い方
```

このように動くはずです:
- エージェントがブラウザを操作して検索する
- 各サイトにアクセスしてスクリーンショットを取得する
- デザインの特徴をレポートとして出力する

### 1.2 トレンドの調査

## Part 2: 構成決定【5分】

### 2.1 リサーチ結果を基に構成を決定

```
リサーチ結果を踏まえて、このLPに必要なセクション構成を提案して。

要件:
- ターゲット: 20-30代のコーヒー好き
- ブランドイメージ: 近未来的 × 温かみ
- ゴール: 来店予約

以下のセクションを含めて:
- ヒーローセクション
- 特徴紹介（3つ）
- メニュー
- アクセス
- CTA（来店予約）

各セクションのキャッチコピーも考えて。
```

## Part 3: ビジュアル生成（Nano Banana）【5分】

### 3.1 ヒーロー画像の生成

```
Nano Banana で、「近未来的なカフェの店内、
ネオンブルーとウォームライト、観葉植物、
4K、フォトリアリスティック」のイメージを生成して。
ファイル名は hero-bg.webp で保存して。
```

### 3.2 アイコン生成

```
アイコン用に、以下の3つのアイコンを生成して:
1. 「コーヒーカップ」- AI抽出の象徴
2. 「植物」- サステナビリティ
3. 「スマホ」- モバイルオーダー

スタイル: ミニマル、線画、白背景
それぞれ icon-coffee.svg, icon-plant.svg, icon-mobile.svg で保存
```

## Part 4: 実装（Skills 活用）【5分】

### 4.1 プロジェクト作成

```
この 06_vibe_coding_advanced フォルダに、
Vite + React + TailwindCSS のプロジェクトを作成して。
プロジェクト名は「ai-coffee-shop」
```

### 4.2 コンポーネント生成

```
以下のコンポーネントを作成して:
- Hero.jsx - ヒーローセクション
- Features.jsx - 特徴紹介（3つのカード）
- Menu.jsx - メニュー一覧
- Access.jsx - アクセス情報（地図埋め込み）
- CTA.jsx - 来店予約ボタン
- Footer.jsx - フッター

デザイン要件:
- グラスモーフィズムスタイル
- ダークモード
- アクセントカラー: #4ecca3
```

### 4.3 Skills を使った自動生成

code-review Skill で、生成されたコードをまとめてチェックしましょう。

```
@code-review ai-coffee-shop/src/components/ のすべてのコンポーネントを
レビューして、改善点があれば自動的に修正して。
```

## 高度なテクニック

### MCP 活用のコツ

```
# Webからデータを取得
「〇〇のAPIドキュメントを読んで、使い方をまとめて」

# 競合分析
「〇〇.comのデザインを参考に、同じようなレイアウトで作って」

# リアルタイム情報
「今日の〇〇の営業時間を調べて、サイトに反映して」
```

### Skills 活用のコツ

```
# 定型作業の自動化
@generate-component Button 「プライマリボタン、青色」

# コードレビュー
@code-review src/components/

# ドキュメント生成
@generate-docs README.md を作成
```

### マルチエージェント連携

```
以下のタスクを並列で実行して:
1. Designer Agent: UIコンポーネントのデザイン
2. Coder Agent: ロジックの実装
3. Researcher Agent: 最新のベストプラクティス調査
```

## 基礎編 vs 発展編の比較

### 開発時間の比較

| 項目 | 従来の開発 | 基礎編 | 発展編 |
|:---|:---:|:---:|:---:|
| リサーチ | 2時間 | - | **10分** |
| デザイン | 4時間 | 20分 | **15分** |
| 実装 | 8時間 | 30分 | **20分** |
| レビュー | 2時間 | - | **5分** |
| **合計** | **16時間** | **50分** | **50分** |

発展編では、リサーチとコードレビューが自動化されるため、より複雑なプロジェクトにも対応できます。
