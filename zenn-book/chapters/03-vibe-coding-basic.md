## このステップのゴール

MCP や Skills を使わずに、シンプルな Vibe Coding を体験します。自然言語で Web ページを作り、Nano Banana で画像を生成するところまで進めましょう。

:::message
所要時間は約20分です。
:::

## Part 1: HTML ページを作る（5分）

### プロジェクトフォルダを作成する

Antigravity のチャットに、こう指示してください。

```
新しいフォルダ「my-first-vibe」を作成して、その中にシンプルなHTMLファイルを作って。
テーマは「自己紹介ページ」で、モダンでおしゃれなデザインにして。
```

`my-first-vibe/` フォルダの中に `index.html` が生成され、CSS も適用されるはずです。

### プレビューを確認する

```
作成したHTMLファイルをブラウザでプレビューして
```

ここでは、まだ何もコードを書いていません。自然言語だけで Web ページが生まれる感覚をつかんでください。

## Part 2: デザインをカスタマイズする（5分）

続けて、対話でデザインを調整していきます。

```
背景色をダークモードに変更して。
アクセントカラーは青緑系（#4ecca3）にして。
```

```
名前の部分にふわっと表示されるアニメーションを追加して
```

```
スマホでも綺麗に見えるようにレスポンシブ対応して
```

一度に複数の変更を頼んでも、AI が順番に対応してくれます。気に入らなければ「元に戻して」と言えば大丈夫です。

## Part 3: Nano Banana で画像を生成する（10分）

Nano Banana は Antigravity に組み込まれた画像生成 AI です。デザイン素材をその場で作れます。

### プロフィール画像を生成する

```
Nano Bananaで、プロフィール用のアバター画像を生成して。
- スタイル: アニメ風またはイラスト風
- 背景: シンプルなグラデーション
- サイズ: 正方形
ファイル名は「avatar.png」で保存して。
```

### 背景画像を生成する

```
Nano Bananaで、ヘッダー用の背景画像を生成して。
- テーマ: 抽象的な波模様
- 色: 紺から青緑へのグラデーション
- 幅広い横長画像
ファイル名は「header-bg.png」で保存して。
```

### 生成した画像を HTML に適用する

```
生成したavatar.pngとheader-bg.pngを自己紹介ページに配置して。
- avatar.pngは丸くトリミングしてプロフィール欄に
- header-bg.pngはヘッダーの背景に
```

## 完成イメージ

```
my-first-vibe/
├── index.html          # メインページ
├── style.css           # スタイルシート
├── avatar.png          # プロフィール画像（Nano Banana で生成）
└── header-bg.png       # 背景画像（Nano Banana で生成）
```

完成したページには、ダークモードのモダンなデザイン、ふわっと表示されるアニメーション、レスポンシブ対応、AI 生成のオリジナル画像が含まれているはずです。

## Vibe Coding がうまくいくコツ

### 具体的に伝える

```
❌ 「かっこいいページを作って」
✅ 「ダークモードで、青緑のアクセントカラー、グラスモーフィズムのカードデザインで作って」
```

ざっくりした指示でも動きますが、具体的に伝えた方が精度は上がります。

### 段階的に進める

```
❌ 「完璧なポートフォリオサイトを作って」
✅ 「まずシンプルな自己紹介ページを作って」→「次にスキルセクションを追加して」
```

最初から完成形を目指さず、少しずつ足していく方がうまくいきます。

### フィードバックを重ねる

```
「もう少し余白を増やして」
「フォントサイズを大きくして」
「もっとシンプルにして」
```

微調整は対話で繰り返せます。「元に戻して」も効きます。


## Human-in-the-Loop: 計画書の歩き方

Vibe Coding で失敗しない最大のコツは、AI が最初に作ってくる「実装計画書（Implementation Plan）」をしっかりレビューすることです。

### 1. Git リポジトリの初期化

計画の最初のステップに **「Initialize Git repository（git init）」** が入っているかを必ず確認してください。これがないと、AI がファイルを消したり壊したりしたときに元に戻せません。もしなければ、「最初に git init を追加して」と指示して計画を書き換えさせましょう。

### 2. 「例外処理」を叩き込む

AI は「ハッピーパス（正常系）」の実装は得意ですが、エラー系は忘れがちです。計画書の段階でツッコミを入れましょう。

- 「通信エラーになったらどうするの？リトライ処理を入れて」
- 「フォームが空のまま送信されたらアラートを出して」
- 「未来の日付は選択できないように制限して」

計画書内の該当箇所をクリックしてコメントを入れるか、チャットで指示を飛ばして、計画書が **Update** されるのを確認してから `Proceed` しましょう。

### 3. 日本語化の徹底

計画書が英語で出てきたら、遠慮なく「日本語に直して」と言いましょう。自分の母国語で確認することで、微妙な認識ズレに気づけるようになります。

## チャレンジ課題

時間に余裕があれば、以下にも挑戦してみましょう。

**初級**: アイコン追加、SNSリンク、フッター
**中級**: ダークモード切替ボタン、スクロールアニメーション、お問い合わせフォーム
**上級**: 複数ページ構成、JavaScript によるインタラクション、ローカルストレージ保存

## トラブルシューティング

**画像が生成されない** → Free プランの月間制限（50枚）に達していないか確認してください。プロンプトは具体的に書くと成功しやすくなります。

**HTML が期待通りにならない** → 指示を具体的にしてみてください。「ボタンの色を #4ecca3 に」のように値を直接伝えると精度が上がります。

**変更が反映されない** → ブラウザのキャッシュをクリアするか、強制リロード（Cmd+Shift+R）を試してください。
