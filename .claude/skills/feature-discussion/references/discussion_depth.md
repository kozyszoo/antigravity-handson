# 議論深さモード

機能の複雑さや重要度に応じて、議論の深さを3段階で調整する。
セッション開始時に自動判定ルールに基づいて自動的に決定する（ユーザーへの確認は不要）。

---

## モード一覧

```
┌──────────────┬─────────────┬──────────────┬──────────────────────┐
│              │   Light     │  Standard   │       Deep           │
├──────────────┼─────────────┼──────────────┼──────────────────────┤
│ 想定ケース   │ 小さな改善   │ 中規模機能   │ 大規模・高リスク機能  │
│ 例           │ ボタン追加   │ フィルター   │ 課金・認証・DnD改修  │
│              │ 表示変更     │ 新しい画面   │ アーキテクチャ変更    │
├──────────────┼─────────────┼──────────────┼──────────────────────┤
│ 参加エージェント│ PM + 1名  │ 全5名        │ 全5名                │
│ R1: チーム議論│ ✅(簡易)     │ ✅(収束まで)  │ ✅(収束まで)          │
│  スコアリング │ 簡易         │ ✅ 全軸      │ ✅ 全軸 + 重み付き    │
│ R2: Devil's  │ ❌ スキップ  │ ✅           │ ✅ + 2巡目           │
│ Quality Gate │ 簡易チェック │ 全項目       │ 全項目 + 厳格モード   │
├──────────────┼─────────────┼──────────────┼──────────────────────┤
│ 推定所要時間  │ 短い        │ 中程度       │ 長い                 │
└──────────────┴─────────────┴──────────────┴──────────────────────┘
```

## Light モード

**用途**: 小さな改善、UI微調整、既存機能の軽微な変更

- **参加エージェント**: PM + 最も関連するエージェント1名
  - UI変更 → PM + Designer
  - 技術的変更 → PM + Engineer
  - ユーザー体験変更 → PM + UX Analyst
- **ラウンド**: R1（チーム議論、収束まで + 簡易スコア）→ R3（統合）
- **R2 Devil's Advocate**: スキップ
- **Quality Gate**: 共通基準のみ、ステップ固有基準は省略

## Standard モード（デフォルト）

**用途**: 中規模の新機能、新しい画面の追加

- **参加エージェント**: 全5名
- **ラウンド**: R1（チーム議論、収束まで + スコアリング）→ R2（Devil's Advocate）→ R3（統合）
- **Quality Gate**: 全項目チェック

## Deep モード

**用途**: 大規模機能、課金・認証・データ構造の変更、高リスクな改修

- **参加エージェント**: 全5名
- **ラウンド**: R1（チーム議論、収束まで + 重み付きスコアリング）→ R2（Devil's Advocate 2巡）→ R3（統合）
- **重み付きスコアリング**: ステップに応じてスコアに重みを設定
  - Step 2: 技術実現性 ×1.5、行動実現性 ×1.2
  - Step 3: ビジネス価値 ×1.5、技術実現性 ×1.2
- **Devil's Advocate 2巡**: 1巡目の指摘への対処案を再度 Devil's Advocate で検証
- **Quality Gate 厳格モード**: 全軸スコア ≥ 3.5、加重平均 ≥ 4.0

## 自動判定ルール

以下の条件に1つでも該当する場合、自動的にモードを引き上げる：

| 条件 | 判定 |
|------|------|
| DB スキーマの変更を伴う | → Standard 以上 |
| 認証・課金に関わる | → Deep |
| 3画面以上の新規追加 | → Standard 以上 |
| 既存APIの破壊的変更 | → Deep |
| ユーザーの発言に「大きな」「重要な」「根本的な」 | → Standard 以上 |

**モード通知フォーマット**（確認不要、通知のみ）:
```
モード: [Standard]（理由: 新しい画面の追加 + DB変更あり）
```
