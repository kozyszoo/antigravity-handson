# 評価基準テンプレート

このテンプレートは knowledge-writer が `evaluation-criteria.md` を生成する際の参考フォーマット。
Analytic Rubric（分析的ルーブリック）形式を採用し、各チェック項目に Severity Rating を付加する。

## フォーマット仕様

```markdown
# {ドメイン名} 評価基準

> **参照フレームワーク**: {Nielsen / WCAG / AIDA / HIG / Material Design / 独自フレームワーク等}
> **形式**: Analytic Rubric（各項目に Severity Rating 付き）
> **バージョン**: {生成日}

## Severity Rating 定義

| Level | 定義 | 影響 |
|-------|------|------|
| **Critical** | これが満たされないと根本的に機能しない | ユーザー離脱 / コンバージョン不成立 |
| **Major** | 品質を大きく下げる観点 | 改善すると高い効果が期待できる |
| **Minor** | 細かい改善点・あれば良い | 体験の洗練度向上 |
```

## Severity 付与ガイドライン

### Critical に設定すべき項目
- ユーザーがアクションを完了できなくなるもの
- 情報の伝達が根本的に失敗するもの
- コンバージョンに直接影響するもの（例: CTA が見えない）
- アクセシビリティ基準（WCAG AA）の必須要件

### Major に設定すべき項目
- ユーザーの理解・信頼に大きく影響するもの
- 改善インパクトが測定可能なもの
- 業界標準から大きく外れているもの

### Minor に設定すべき項目
- 洗練度・完成度に関わるもの
- ベストプラクティスとして「あれば良い」もの
- ニッチな状況でのみ影響するもの

## 定量基準の記述例

以下のような定量的記述を推奨（曖昧な記述は避ける）:

| 曖昧（避ける） | 定量的（推奨） |
|--------------|--------------|
| 「コントラストが十分か」 | 「コントラスト比 4.5:1 以上（WCAG AA）」 |
| 「テキストが読みやすいか」 | 「本文フォントサイズ 16px 以上、行間 1.5em 以上」 |
| 「CTAが目立つか」 | 「CTA のコントラスト比 3:1 以上、タップ領域 44×44pt 以上」 |
| 「ロードが速いか」 | 「LCP 2.5秒以下（Core Web Vitals）」 |
| 「情報が整理されているか」 | 「ファーストビューに価値命題・CTA・信頼要素が全て含まれる」 |
