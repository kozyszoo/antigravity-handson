# UI レビュー評価基準 詳細リファレンス

各エージェントが参照する詳細な評価基準。SKILL.md から必要に応じて読み込む。

## コントラスト比の計算ガイド

### WCAG 2.1 AA 基準
| 要素 | 最低比率 |
|------|---------|
| 通常テキスト（<18px bold / <24px regular） | 4.5:1 |
| 大きいテキスト（≥18px bold / ≥24px regular） | 3:1 |
| UIコンポーネント、アイコン | 3:1 |
| 非活性要素 | 基準なし（ただし3:1推奨） |

### よくあるNG パターン
- ライトグレーテキスト on 白背景（#999 on #FFF → 2.85:1 NG）
- プレースホルダーの薄すぎるグレー
- カラーボタン上の白テキスト（背景が明るすぎ）

## タッチターゲットサイズ

### プラットフォーム別基準
| プラットフォーム | 最低サイズ | 推奨サイズ |
|----------------|-----------|-----------|
| iOS | 44x44pt | 44x44pt |
| Android | 48x48dp | 48x48dp |
| Web (WCAG) | 24x24px | 44x44px |

### 例外が許容されるケース
- インラインテキストリンク（ただし十分な行間が必要）
- テーブルのセル内アクション（行全体がタップ領域の場合）

## カラーパレット設計ガイド

### 60-30-10 ルール
- 60%: ベースカラー（背景、大きな面積）
- 30%: セカンダリカラー（カード、セクション区切り）
- 10%: アクセントカラー（CTA、重要な要素）

### モバイルアプリでよく使われるパターン
- Light mode: 白背景 + グレー系 + ブランドカラー
- Dark mode: #121212背景 + #1E1E1E面 + ブランドカラー

## スペーシングシステム

### 4px ベースグリッド
- 4px: 最小間隔（アイコンとラベル間）
- 8px: 要素内パディング
- 12px: 関連要素間
- 16px: セクション内の要素間
- 24px: セクション間
- 32px: 大きなセクション間
- 48px: ページセクション間
