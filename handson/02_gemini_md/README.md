# 📝 Hands-on 02: Rules（GEMINI.md）設定

## 目標
プロジェクト固有のルールをエージェントに教える設定ファイルを作成する。

---

## 📖 Rules とは？

**Rules** は、エージェントの振る舞いを制御する**持続的な指示**です。
コーディングスタイル、言語設定、プロジェクト固有の規約などを定義します。

> 📖 公式ドキュメント: https://antigravity.google/docs/rules-workflows

### 設定ファイルの種類

| ファイル名 | 説明 |
|:---|:---|
| `GEMINI.md` | プロジェクトルートに配置。最も一般的 |
| `AI_INSTRUCTIONS.md` | 別名。同じ機能 |
| `ANTIGRAVITY.md` | 別名。同じ機能 |

### 配置場所と適用範囲

| レベル | パス | 適用範囲 |
|:---|:---|:---|
| **グローバル** | `~/.gemini/GEMINI.md` | すべてのプロジェクトに適用 |
| **ワークスペース** | `.agent/rules/` フォルダ内 | そのワークスペースにのみ適用 |
| **プロジェクトルート** | `GEMINI.md` | そのプロジェクト全体に適用 |
| **サブディレクトリ** | 各フォルダ内の `GEMINI.md` | そのフォルダ以下にのみ適用 |

### Activation Mode（ルールの発動条件）

Rules には **4つの発動モード** があります：

| モード | 説明 | 例 |
|:---|:---|:---|
| **Always On** | 常に有効 | コーディング規約、言語設定 |
| **Manual** | `@ファイル名` で明示的に呼び出し | 特定のレビュー基準 |
| **Model Decision** | AIが自然言語の説明から自動判断 | 関連性のあるルール |
| **Glob** | 特定のファイルパターンに一致時に発動 | `src/**/*.ts` 等 |

## ✅ ハンズオン: GEMINI.md を作成しよう

### Step 1: ファイル作成

プロジェクトルートの `GEMINI.md` を開いて確認してください（既に作成済み）。

### Step 2: 基本テンプレート

以下が GEMINI.md の基本構成です：

```markdown
# プロジェクト設定 - Google Antigravity Hands-on

## 基本ルール
- 常に**日本語**で応答してください。
- コードには必ず**コメント**を入れてください（日本語可）。
- エラーが発生した場合は、原因と解決策を提示してください。

## 技術スタック
- HTML / CSS / JavaScript（Vanilla）
- Node.js (任意)

## コーディング規約
- 変数名: camelCase
- 関数名: camelCase
- ファイル名: kebab-case（例: `my-component.js`）
```

### Step 3: 動作確認

Antigravity のチャットで以下を試してください:

```
「Hello World」を表示する HTML ファイルを作成して。
```

**確認ポイント**:
- [ ] 日本語で応答されているか
- [ ] コードにコメントが入っているか
- [ ] 変数名が camelCase になっているか

## 🎯 応用課題

### 課題 1: ルールを追加してみよう

`GEMINI.md` に以下を追加して、エージェントの振る舞いの変化を確認：

```markdown
## 追加ルール
- ファイル作成時は必ず確認を求めてください。
- 長いコードは分割して説明してください。
```

### 課題 2: Workflows を体験しよう（任意）

**Workflows** は、繰り返しのタスクを自動化する仕組みです。

1. `.agent/workflows/` フォルダに `.md` ファイルを作成
2. チャットで `/ワークフロー名` と入力して実行

```markdown
# .agent/workflows/hello.md

---
description: 挨拶のテスト用ワークフロー
---

1. ユーザーに「こんにちは！何をお手伝いしましょうか？」と挨拶する
2. 現在の日付と時刻を表示する
3. 現在のディレクトリ構造を簡単に説明する
```

実行方法:
```
/hello
```

## 📌 Tips

| 設定項目 | 効果 |
|:---|:---|
| `常に日本語で` | 応答言語の固定 |
| `コメントを入れて` | 可読性の向上 |
| `技術スタック明記` | 不要なライブラリ提案を抑制 |
| `ディレクトリ構造` | ファイル配置の一貫性 |
| `Glob パターン` | ファイル種類ごとにルールを適用 |
