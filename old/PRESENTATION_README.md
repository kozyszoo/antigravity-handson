# 📊 Google Antigravity 登壇資料

Marpを使用したAntigravityハンズオンの登壇用プレゼンテーション資料です。

## 📁 ファイル構成

- `presentation.md` - Marpソースファイル
- `presentation.html` - HTML版プレゼンテーション
- `presentation.pdf` - PDF版プレゼンテーション

## 🎨 デザインの特徴

- **ダークテーマ**: 目に優しい深い青と緑のグラデーション
- **視覚的な強調**: 重要なポイントをハイライト表示
- **構造化されたレイアウト**: 2カラムレイアウト、表、コードブロック
- **カラースキーム**:
  - メインカラー: `#4ecca3` (ターコイズグリーン)
  - アクセントカラー: `#93deff` (スカイブルー)
  - 強調色: `#ffd700` (ゴールド)

## 🚀 使い方

### プレゼンテーションの表示

#### HTML版（推奨）
```bash
open presentation.html
```

ブラウザで開くと、以下の操作が可能です:
- **→** または **Space**: 次のスライド
- **←**: 前のスライド
- **F**: フルスクリーンモード
- **Esc**: フルスクリーン解除

#### PDF版
```bash
open presentation.pdf
```

### 編集と再生成

1. `presentation.md` を編集
2. 以下のコマンドで再生成:

```bash
# HTML生成
marp presentation.md -o presentation.html --html

# PDF生成
marp presentation.md -o presentation.pdf --html --allow-local-files
```

### ライブプレビュー（開発時）

```bash
marp -s presentation.md
```

ブラウザで `http://localhost:8080` を開くと、リアルタイムプレビューが表示されます。

## 📋 プレゼンテーション構成

### 全体構成（約30スライド）

1. **導入** (5分)
   - タイトル
   - アジェンダ
   - 課題提起

2. **Antigravity概要** (10分)
   - 基本概念
   - 従来ツールとの比較
   - 主要機能

3. **技術詳細** (15分)
   - Agent-First Architecture
   - Dual View System
   - Multi-Model Support
   - MCP、Skills

4. **実践デモ** (20分)
   - Vibe Coding 5ステップ
   - 実際の開発フロー

5. **まとめ** (5分)
   - 強み総括
   - 学習リソース
   - Q&A

## 🎯 登壇時のポイント

### スライド1-5: 導入
- **聴衆の共感を得る**: 「こんな経験ありませんか?」で開発の課題を共有
- **期待感を高める**: Antigravityがどう解決するかを示唆

### スライド6-15: 概要と比較
- **差別化を明確に**: 従来ツールとの比較表で視覚的に訴求
- **哲学を伝える**: 「重力からの解放」というコンセプトを強調

### スライド16-22: 技術詳細
- **具体例を交える**: 各機能の実用例を説明
- **質問を促す**: 技術的な疑問点がないか確認

### スライド23-28: Vibe Codingデモ
- **実際に操作**: 可能であればライブデモを実施
- **ステップを明確に**: 各段階で何が起きているか説明
- **時間短縮を強調**: 従来3.5時間 → 10分の衝撃

### スライド29-32: まとめ
- **行動を促す**: 今日から始められることを伝える
- **リソース提供**: 学習の次のステップを示す

## 💡 デモ準備

## 🎤 プレゼンテーションTips

### 時間配分
- **導入**: 5分
- **概要**: 10分
- **技術詳細**: 15分
- **デモ**: 20分
- **まとめ**: 5分
- **Q&A**: 15分

### 話し方
- **ゆっくり明瞭に**: 専門用語は特に丁寧に
- **聴衆を見る**: スライドばかり見ない
- **質問を歓迎**: 途中でも質問OKと伝える

### トラブル対応
- **デモ失敗時**: 録画映像に切り替え
- **時間超過時**: デモをスキップして概要のみ
- **質問多数時**: 後でSlackやメールで対応
